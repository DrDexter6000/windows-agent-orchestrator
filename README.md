# Windows Agent Orchestrator (WAO)

Windows-native, headless, runtime-agnostic orchestrator for local agent runtimes
(claude-code / codex / opencode-serve). Drives agents via subprocess or HTTP,
records everything to JSONL transcripts, provides git worktree isolation, resume,
token/cost metrics, declarative DAG workflows, and evidence-chain scorecard gating.

- **Zero dependencies** — Node ESM, no `npm install`, no Docker/WSL.
- **Transcript is source of truth** — every run reconstructable from `runs/<runId>.jsonl`.
- **Windows-native** — worktree isolation + process-tree cleanup tuned for Windows.

## Vision

WAO is a Windows-native control plane that lets a lead agent dispatch certified worker
agents to do real repository work. It keeps orchestration out of worker system prompts:
workers receive normal task prompts, while WAO owns transcripts, state, isolation,
workflow execution, metrics, and evidence-chain scorecard gates.

## Current Status

WAO is ready for **supervised production dispatch trial**. It can be used today by a
lead agent to run real repo research, coding, bug fixing, documentation, tests,
daemon-backed workflow orchestration, and scorecard-gated verification.

Implemented (M0–M8 + fresh-runtime hardening): state machine, multi-backend
(opencode-serve + claude-code + codex), worktree isolation, resume, metrics
aggregation, declarative DAG engine, parameterized workflow templates, daemon
supervision, scorecard evidence gating, runtime certification, diagnostics, cost
forecasting, and safe Claude Code provider wrappers.

Current dispatch policy is **Claude Code-first** for real coding/tool-use tasks.
Use opencode only as an optional low-cost research / long-context lane unless a
fresh certification proves the exact runtime+model is suitable for the task.
Important boundary: opencode is still a fallback lane, but its stop path now has
bounded backend quietness verification (`run.stop_verified` / `run.stop_unverified`)
and TD-37/TD-38 are tracked in `docs/tech-debt.md`. The runtime truth source is
`runs/reliability-summary.json` after `npm run reliability`.

See [`docs/roadmap.md`](docs/roadmap.md) for milestone status + current test counts,
and [`docs/tech-debt.md`](docs/tech-debt.md) for the open tech-debt register.

## Quick start

```powershell
git clone https://github.com/DrDexter6000/windows-agent-orchestrator.git D:\projects\windows-agent-orchestrator-poc
cd D:\projects\windows-agent-orchestrator-poc
# no npm install — zero dependencies

# 1. Configure the agent registry (copy template, edit cwd + serveUrl)
Copy-Item config/agents.example.json config/agents.json

# 2. (opencode-serve backend only) start the serve
#    Use scripts/serve.ps1 — it injects provider API keys (ZHIPU/KIMI) from the
#    User registry. A bare `opencode serve` started without those env vars will
#    401 on every provider call (WAO fast-fails on it, but the run still fails).
powershell -ExecutionPolicy Bypass -File scripts/serve.ps1 -Port 4297

# 3. Verify
# registry list = inventory + certification status; registry validate = static schema; registry check = live opencode health
npm run cli -- registry list --registry config/agents.json
npm run cli -- registry check --registry config/agents.json

# 4. Run a certified worker
npm run cli -- run coder_low --prompt "Summarize the README"
```

Node ≥ 22 required (`node --version`).

## Commands (overview)

```powershell
npm run cli -- run <agentId> --prompt "..."           # run + wait, prints result
npm run cli -- spawn <agentId> --prompt "..."         # fire-and-forget
npm run cli -- workflow run <file.mjs> [--vars k=v]   # declarative DAG
npm run cli -- runs metrics <runId>                   # tokens / cost
npm run cli -- runs scorecard <runId>                 # evidence gate result
npm run cli -- status|tail|collect|stop|retry|resume <runId>
```

Full command reference: `npm run cli -- help`, or [`SKILL.md`](SKILL.md).

## Testing

```powershell
npm test            # all unit/integration tests (mock subprocesses, no API tokens)
npm run smoke       # real CLI smoke (claude/codex/opencode — consumes API tokens)
npm run reliability # runtime/model certification matrix — consumes API tokens
```
